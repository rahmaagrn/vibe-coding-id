## Istilah Dasar

### AI (Artificial Intelligence)
Kecerdasan buatan — program komputer yang bisa berpikir dan belajar seperti manusia. Contoh: Claude, ChatGPT, Gemini.

### Prompt
Perintah atau deskripsi yang kamu kasih ke AI. Semakin jelas prompt kamu, semakin bagus hasilnya.

### Vibe Coding
Cara membuat software dengan ngomong ke AI apa yang kamu mau, dan AI yang nulis kodenya.

### Model AI
"Otak" dari AI — versi dan jenis AI yang berbeda punya kemampuan berbeda. Contoh: Claude Opus, Claude Sonnet, GPT-4, DeepSeek.

## Istilah Programming

### Kode / Code
Tulisan instruksi yang dibaca komputer. Dulu harus ditulis manual, sekarang bisa dihasilkan oleh AI.

### Repository / Repo
Tempat menyimpan semua file proyek kamu. Seperti folder di komputer, tapi online di GitHub.

### Framework
Kerangka kerja yang bikin coding lebih cepat dan terstruktur. Bayangkan seperti template bangunan — kamu tinggal isi bagian dalamnya.

### Library / Pustaka
Kumpulan kode siap pakai yang bisa kamu gunakan di proyekmu. Seperti "bahan siap pakai" daripada masak dari nol.

### Syntax
Aturan penulisan dalam bahasa pemrograman. Seperti grammar di bahasa Indonesia — kalau salah tulis, komputer bingung.

### Bug
Kesalahan di kode yang bikin program tidak jalan seperti yang diharapkan.

### Debug
Proses mencari dan memperbaiki bug. Di vibe coding, kamu tinggal bilang ke AI "ini error-nya, tolong fix."

## Istilah Build & Deploy

### Build
Proses mengubah kode menjadi aplikasi yang bisa dijalankan. Seperti memasak — dari bahan mentah jadi makanan siap santap.

### Deployment
Proses bikin aplikasi kamu bisa diakses orang lain secara online. Dari komputer kamu → ke internet.

### Hosting
Layanan yang nyimpen aplikasi kamu di server (komputer khusus) supaya bisa diakses 24 jam. Contoh: Vercel, Railway, Netlify, GitHub Pages.

### Development
Tahap membuat aplikasi — masih di komputer lokal, belum online.

### Production
Tahap aplikasi sudah live dan dipakai orang sungguhan.

## Istilah Proyek

### MVP (Minimum Viable Product)
Produk paling sederhana yang masih berguna — cukup untuk testing apakah ide kamu layak. Bukan versi sempurna, tapi versi "cukup jalan."

### Prototipe / Prototype
Versi percobaan dari ide kamu. Biasanya dibuat cepat untuk testing dan dikasih lihat ke orang lain.

### Iterasi
Proses bikin sesuatu → dikasih feedback → diperbaiki → bikin lagi. Loop ini adalah inti dari vibe coding.

### Open Source
Proyek yang kodenya bisa dilihat, dipakai, dan dikontribusi oleh siapa saja secara gratis.

## Istilah AI Coding

### Context Window / Konteks
"Memori jangka pendek" AI — seberapa banyak informasi yang bisa diingat AI dalam satu sesi obrolan. Makin besar, makin baik untuk proyek besar.

### Token
Unit terkecil yang dibaca AI. 1 token ≈ 3-4 huruf dalam bahasa Inggris. Biaya pakai AI dihitung per token.

### Hallucination / Halusinasi
Saat AI menjawab dengan percaya diri tapi salah. Ini terjadi dan wajar — selalu cek hasil AI sebelum dipakai.

### Agent
AI yang bisa melakukan tugas multi-langkah sendiri — bukan cuma jawab pertanyaan, tapi bikin rencana dan eksekusi.

## Istilah GitHub

### GitHub
Website untuk nyimpen dan manage kode secara online. Seperti Google Drive, tapi khusus untuk kode.

### Fork
Meng-copy repo orang lain ke akun GitHub kamu sendiri. Biar kamu bisa modifikasi tanpa mengubah repo aslinya.

### Pull Request (PR)
Permintaan untuk menggabungkan perubahan kode kamu ke repo asli. Cara utama berkontribusi di open source.

### Issue
Laporan masalah, saran fitur, atau diskusi di sebuah repo GitHub.

### Star
Like/bookmark untuk repo GitHub. Cara orang menunjukkan "ini proyek keren" tanpa harus berkontribusi langsung.

## Istilah Startup

### Startup
Perusahaan baru yang bertujuan tumbuh cepat dengan menyelesaikan masalah yang belum ada solusi baiknya.

### Founder
Orang yang memulai perusahaan. Bisa teknis (bisa coding) atau non-teknis.

### Pivot
Saat startup mengubah arah produknya karena ternyata solusi awal tidak cocok dengan pasar.

### Scale
Tahap startup mulai tumbuh cepat setelah menemukan product-market fit.

### Product-Market Fit
Kondisi dimana produk kamu benar-benar dibutuhkan dan diinginkan pasar. Orang pakai, bayar, dan merekomendasikan.
